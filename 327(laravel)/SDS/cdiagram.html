Detailed Class Diagram

In the above, a simplified view of the model, view and controller classes shown with UML class diagrams. View classes have been clearly separated from the model and controller. Controller exchanges data between the model and view when using the User_Action class. TO-LET_Viewer class is the viewer and User and House are the model classes which holds data.
