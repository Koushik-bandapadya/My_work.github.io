3. Description of Design Components:

The TO-LET service mainly consists of three major components, namely Front end, Back end and API.

Client side:

• The user component explains about how the client is going to deal with the site, how it’s going to work, what information the user has to provide in terms of accessing the site.

• Once a customer searches the TO-LET site they will be provided with a log in page. They have to insert the username and password for authentication and can only enter the site once the access is accepted. Sign up option is also there for a new user.

• All the information provided by a user will be archived by the admin by a control panel.


• A user logged in into the site can see the available advertisements of properties. They can post an advertisement and can also search categorically. A map view can also be achieved to see any available properties near the customer.


Server side:

• The system will set the advertisements in categories, it will search for the items searched by the user in the library and show results, it will show a map view if the user desires (using API).


• The system will make an entry for every individual user in the library and recognize them for safety purpose and will cross match it once a user tries to log in for the sake of verification.
